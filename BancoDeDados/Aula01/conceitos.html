Fundamentos de Bancos de Dados

O Alicerce da Era Digital: Da Teoria à Aplicação Prática

O Que é um Banco de Dados?

Um banco de dados é uma coleção de dados organizada sistematicamente e armazenada de forma eletrônica. Ele foi projetado para facilitar o gerenciamento e o acesso aos dados, garantindo sua organização, segurança e disponibilidade, mesmo à medida que a coleção de dados cresce em volume e complexidade. É a estrutura fundamental que permite que dados sejam armazenados, organizados, acessados e protegidos, transformando a matéria-prima digital em um recurso estratégico e valioso.

Dados vs. Informação

D

O Que São Dados?

Dados são a matéria-prima, o registro bruto e inespecífico de fatos ou eventos. Em sua forma original, um dado não carrega um significado completo por si só. Ex: "1985", "Maria Silva".

I

O Que é Informação?

A informação é o resultado do processamento e da contextualização dos dados. É quando os dados adquirem um significado detalhado e são trabalhados para agregar valor. Ex: "Maria Silva nasceu em 1985".

Diferença e Dependência: A diferença crucial reside no significado e no contexto. Dados são inespecíficos, enquanto a informação é específica e útil. A informação não pode existir sem os dados, e os dados só se tornam valiosos quando transformados em informação.

Quem Deveria Usar e Onde São Usados?

O uso de bancos de dados é onipresente, permeando praticamente todos os setores da economia e da sociedade. Profissionais como o Administrador de Banco de Dados (DBA) e o Analista de Dados são responsáveis por seu gerenciamento, mas seu uso final se estende a todos que dependem das informações que eles contêm.

Bancos de Dados Relacionais e SGBDs

O Que é um Banco de Dados Relacional?

É o tipo de banco de dados mais comum, que organiza dados em tabelas (ou "relações") com colunas e linhas. Essa estrutura facilita a visualização e a compreensão de como diferentes dados se conectam. A linguagem de programação padrão para interagir com eles é o SQL.

O Que São SGBDs?

SGBD (Sistema de Gerenciamento de Banco de Dados) é o software responsável por gerenciar e controlar as bases de dados. Ele atua como um intermediário entre a aplicação e os dados, abstraindo a complexidade de acesso e organização.

Principais SGBDs:

  • Oracle Database
  • MySQL
  • SQL Server
  • PostgreSQL
  • MongoDB (Não relacional)

Projeto e Ciclo de Vida

O Projeto de Banco de Dados

É a representação dos objetos do banco de dados, seguindo três fases:

  • Modelagem Conceitual: Representação abstrata dos requisitos do negócio (o que).
  • Modelagem Lógica: Tradução do modelo conceitual para um modelo estruturado (como o relacional).
  • Modelagem Física: Implementação do esquema lógico no SGBD, considerando os detalhes técnicos.

Ciclo de Vida de um Sistema de Banco de Dados

É um ciclo contínuo de gestão e evolução, composto por 6 fases:

  • Estudo Inicial
  • Desenho
  • Implementação e Carregamento
  • Teste e Avaliação
  • Operação
  • Manutenção e Avaliação

Modelagem de Dados e MER

A Modelagem de Dados é a "planta baixa" que ajuda a visualizar e entender os relacionamentos entre os dados. O Modelo Entidade-Relacionamento (MER) é uma técnica que representa o mundo real de forma abstrata.

Entidade

Chaves

Atributos que identificam e conectam os dados.

  • Chave Primária (PK): Identifica um registro de forma única.
  • Chave Estrangeira (FK): Conecta tabelas, referenciando a PK de outra tabela.

SQL: A Linguagem dos Dados

SQL (Structured Query Language) é a linguagem padrão para interagir com bancos de dados relacionais. É uma linguagem declarativa que permite descrever o resultado desejado, tornando o acesso aos dados flexível e poderoso.

Principais Comandos SQL

Linguagem de Definição de Dados (DDL)

  • CREATE TABLE: Cria uma nova tabela.
  • ALTER TABLE: Modifica a estrutura de uma tabela.

Linguagem de Manipulação de Dados (DML)

  • INSERT INTO: Adiciona novos registros.
  • UPDATE: Modifica dados existentes.
  • DELETE FROM: Remove registros.
  • SELECT: Recupera dados (o mais utilizado).
